## 1. 简介
> 官方 CRM Demo 已预置常用 AI 员工与任务，无需配置、快速体验。

**AI 员工**是内置在 NocoBase 界面的"角色化助手"。根据打开位置的不同，他们能理解你所在页面与当前数据（如 Leads 列表、Accounts 详情、表单页），并直接完成分析、提取、翻译、检索、配置建议等工作。


### 1.1. 打开位置
在官方 CRM Demo 中，你最常与他们以这几种方式相遇：

* **右下角全局面板**：随时呼出，并带着当前页面上下文继续追问或咨询配置问题。

![AI员工全局面板](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-36-54.png)

* **区块右上角的操作按钮**：例如在 **Leads** 列表中点击 **Viz**，即可得到趋势图、阶段漏斗和简要结论（无需写提示词）。
* ![01getting-started-2025-10-27-00-41-34](https://static-docs.nocobase.com/01getting-started-2025-10-27-00-41-34.png)

* **特定界面**: 如代码编辑器中的前端工程师 **Orin** ，数据源配置页右下角的建模助手 **Nathan**
![01getting-started-2025-10-27-00-42-28](https://static-docs.nocobase.com/01getting-started-2025-10-27-00-42-28.png)
![01getting-started-2025-10-27-00-43-12](https://static-docs.nocobase.com/01getting-started-2025-10-27-00-43-12.png)


### 1.2. 设计理念: 角色 & 场景

每位 AI 员工都由两部分组成：
* **稳定的角色**：他们的"性格与专业"，如 Viz 的分析能力、Lexi 的语言天赋，这部分是长期固定的。
* **灵活的任务**：根据业务需要随时调整的任务脚本或工具，如"分析本月销售线索""翻译当前表单"。

这种设计让 AI 员工既保持一贯的风格，又能快速适配不同业务场景。

## 2. 场景和使用方式

> 建议从"预置任务"开始，先看见结果再研究细节。

### 2.1 查看线索趋势与阶段

* **入口**：Leads 列表 → 区块右上角 **Viz**
* **操作**：选择预置任务（如"阶段转化与趋势""来源渠道对比""本月 vs 上月盘点"）→ 等待图表与 2–3 条要点解读

![多任务选择](https://static-docs.nocobase.com/00_QuickStart_cn-2025-09-29-16-46-00.png)

* **继续深入**：可追问"只看华东/按周汇总/排除自营渠道"等
* **说明**：Viz 默认"**一图一重点**"，若数据不足会明确提示而不编造结论；图表采用规范的 ECharts JSON 输出。

### 2.2 把聊天/邮件变成表单数据

* **入口**：Leads → 新建表单 → 区块下方操作区 **Dex / Lexi**

![01getting-started-2025-10-27-00-32-51](https://static-docs.nocobase.com/01getting-started-2025-10-27-00-32-51.png)

* **操作**：

  * Dex：粘贴原始文本 → 执行"提取并填充"，自动识别客户名/联系方式/问题类型等并写入字段
  * Lexi：执行"翻译当前表单内容"，保持数值与日期格式不变
* **说明**：若内容缺失会标注，方便人工快速补齐后提交。

### 2.3 获取客户公司概况

* **入口**：某个 Accounts 详情 → 区块右上角 **Vera**
![01getting-started-2025-10-27-00-35-11](https://static-docs.nocobase.com/01getting-started-2025-10-27-00-35-11.png)
* **操作**：选择"公司背景与行业概况"任务 → 联网检索 → 输出摘要与要点清单
* **继续深入**：可展开"近期动态/竞品对比/潜在风险"。

### 2.4 碰到配置或权限问题

* **入口**：右下角全局面板 → **Cole**
* **操作**：描述现象（如"按钮不显示""谁能访问这个页面"）→ 获得检查路径与处理建议
* **说明**：Cole 给出诊断与操作路径，不直接改你的数据或权限。

## 3. AI 员工一览与速查

> 本节集中给出**完整清单 + 快速去向**，避免与正文示例重复。

| AI员工     | 职能定位   | 核心场景                       | 适用情况               |
| ---------- | ---------- | ------------------------------ | ---------------------- |
| [**Viz**](/ai-employees/built-in/viz)    | 洞察分析师 | 经营分析、报表生成、趋势洞察   | 需要从数据中获得洞察   |
| [**Orin**](/ai-employees/built-in/data-modeling)   | 数据建模师 | 建表改表、关系设计、索引优化   | 设计或优化数据库结构   |
| [**Dex**](/ai-employees/built-in/form-filler)    | 数据整理助手 | 文本提取、表单填充、信息结构化 | 处理杂乱的非结构化信息 |
| **Lexi**   | 翻译专家   | 多语言翻译、本地化、术语管理   | 跨语言沟通和文档处理   |
| [**Nathan**](/ai-employees/built-in/ai-coding) | 前端工程师 | 字段联动、组件开发、表单逻辑   | 实现前端交互和业务逻辑 |
| **Cole**   | NocoBase助手   | NocoBase 配置指导、最佳实践     | "是什么/怎么用/为什么"     |
| **Vera**   | 研究分析师 | 竞品分析、行业研究、情报收集   | 需要外部信息和市场洞察 |

**使用小贴士**

* **先看页面区块**：多数页面已有"员工按钮"，优先使用（已绑定上下文与预置任务）。
* **从模板任务开始**：先跑通一个可见结果，再主动提出第二个问题。
* **说清数据范围**：如"本月/本周、仅第三方渠道"，Viz 会更快、更准地出图。
